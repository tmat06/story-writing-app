## Decision
approve

## Differentiation thesis
This ticket is worth approving because it moves us beyond generic "has AI" parity into trust-by-design: explicit consent gates, per-story controls, and visible AI activity history. Living Writer sets the bar for optional AI, but this scope can stand out by making consent and control more granular and auditable in day-to-day writing flow.

## Competitor evidence
- Living Writer positions AI as an optional capability and frames privacy/ownership as a core value in product messaging: https://livingwriter.com/ai-features/
- Living Writer Terms include dedicated AI Usage terms (including optional use and model/ownership constraints), showing this is already a competitive decision area: https://livingwriter.com/terms-conditions/
- Campfire’s FAQ references separate paths around Campfire/Blaze and a broad write/read/share ecosystem, but does not foreground per-story AI consent controls in the core writer flow: https://www.campfirewriting.com/faq

## Risks / trade-offs
- Added consent prompts can reduce first-use conversion if copy is verbose or interruptive.
- Per-story + global toggles can create state confusion without clear precedence rules.
- Activity logging adds UX and engineering complexity; value drops if logs are too technical for writers.

## Recommendation
Approve with design emphasis on clarity and low friction: default-off AI, one-tap explicit opt-in, clear global-vs-story precedence, and human-readable activity entries. This is a positive differentiator versus feature-parity AI checkboxes.
